接口概要说明

接口传输协议

本项目中的所有接口都是基于HTTP(s)协议传输数据
HTTP协议

接口命名规范

  • 接口统一使用Rest Api的风格，如
    新增用户:post /user
    删除用户:delete /user/1
    修改用户:put /user/1
    获取用户信息:get /user/1
    获取用户列表:get /users
    查询用户:get /users?name=XXX

  • 接口定义应该简洁，能使用小写尽量使用小写

  • 接口统一以/rest/开头，参考以下格式：
    /rest/{对象缩写}/{对象ID}
    /rest/{对象缩写}/{对象ID}/{内嵌对象}/{内嵌对象ID}
    /rest/{对象缩写}/{对象动作}
    /rest/{对象分类}/{对象缩写}/{对象ID}

接口格式规范

  • 返回值格式
    本项目中所有接口都会以下如的格式返回数据：

    {
   code:'',    //状态码
   message:'', //消息字段，如果出现业务错误，返回业务错误的具体信息
   data:''     //业务数据字段
}
  • 返回码格式
    业务错误统一返回 

    非业务错误根据当前情况返回对应的HTTP状态码,如
    401,404```。

用户

用户登录-密码登录

接口描述

用户登录-手机号密码

请求路径

post /rest/user/loginpwd

请求格式

  • application/json

是否需要登录

否

访问限制

暂无

请求字段说明

参数名 是否必选 类型 说明 默认值

| phone | 是 | String | 手机号 | - |
| password | 是 | String | 用户密码 | - |

返回结果

{
    "code":200,
    "message":"",
    "data":{
        "account":"abc123",
        "name":"游客_sadas",
        "phone":"15218320000",
        "email":"41222@qq.com",
        "sex":"1",
        "picture":"http:dioaiosjdi.jpg", 
        "level":"1",
        "createTime":"2017-01-01",
        "status":"1",
        "merchant":"0"
    }
}

返回字段说明

参数名 类型 说明
account String 用户帐号
name String 用户姓名
phone String 手机号码
email String 邮箱
sex number 性别
picture String 头像
level number 等级
createTime String 注册日期
status number 客户状态0–失效，1–生效
merchant number 商家标记0–未申请商家身份，1–已申请商家身份

错误码说明

错误代码 错误标识 说明
400 parameter_error 手机号格式错误
401 invalid_password 密码错误
